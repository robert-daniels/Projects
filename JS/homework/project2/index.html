  1. At the end of the Browser environment lesson, the author makes this comment:
    "Now we'll get down to learning DOM, because the document plays the central role in the UI."
    Explain this sentence to a non-technical person. What is the DOM and how is it created? What is the document? What is the UI? How does this relate to JavaScript?  

    The DOM (Document Object Model) is "built" by the browser as it reads in the HTML file. The webpage is envisioned as a document with each item in that document being an object. In Object Oriented Programming, objects have their own properties and methods. These objects are considered independent from each other. (They may be able to interface through their methods, though.)

    The compilation of these objects creates a user interface (UI) visually to the user that (in theory) they can easily comprehend and interact with.

    JavaScript can call the properties and methods of these objects and manipulate them. By doing this, the webpage instance ceases to be static and can be altered by the end user even if they don't have access to the underlying HTML file on the server.

  2. What specification documents does the author recommend (include the link and the description of the specification in your answer.)

    The author recommends the following:

    • The DOM Living Standard

      Per the standard, this defines a platform-neutral model for events, aborting, and node trees. Properties and Methods of the various DOM objects are described here.

    • The CSSOM: CSS Object Model

      This repository explains CSS rules and stylesheets as represented by objects. This also goes into detail on how to read and write with these objects.

    • What, in general, does a property of an object do?

      A property of an object is a value that is unique to that instance of the object. For example, if there were two "studentJCCC" objects student1 might have creditHours = 15 and student2 might have creditHours = 12. The property is creditHours, but each object has a different value.
